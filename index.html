Glossary

This tab explains the main terms used across the app so the simulator is easier to follow for beginners.

Tiling

Breaking a large matrix operation into smaller blocks that can be reused in faster on-chip memory.

Shared memory

On-chip memory that threads in the same block can reuse cooperatively.

Register pressure

How much per-thread register use is squeezing occupancy or limiting how many warps can stay active.

Occupancy

How many warps are resident and ready to run on a multiprocessor at once.

Tensor core / MMA

Specialized hardware path for matrix multiply-accumulate operations when layout and datatype requirements are met.

INT4 quantization

Representing values with 4-bit integers to reduce memory cost and bandwidth, usually with some accuracy tradeoff.

Pipeline depth

How many stages of data movement and compute are overlapped to hide latency.

Measured vs simulated

Simulated values come from the model in this app; measured values would come from real benchmark or profiler output.

Use Cases

GPU Optimization Explorer is useful when you need to explain, compare, or communicate optimization choices without jumping straight into raw kernel code or profiler tables.

Teaching and learning

Use it to explain tiling, tensor-core paths, register pressure, and quantization to students or mixed technical audiences.

Hackathon demos

Use Story and Explore together to show the problem, the optimization path, and the resulting tradeoffs in a short live demo.

Team discussions

Use it to compare candidate optimization directions before committing engineering time to benchmarking and implementation.

Benchmark interpretation

Use the future Profile Data tab to compare measured profiler exports or benchmark CSVs against the simulator expectations.

Research communication

Use it in office hours, project reviews, or presentations when you need a visual way to discuss performance behavior.
